## 1. 項目一句話

用共振結構生成有情緒、有世界觀、可直接放進遊戲使用的小音效與短聲音素材。

更完整地說：

> 這是一個開源的情緒驅動音效生成工具。使用者用情緒、場景、材質或聲音意圖描述想要的效果，系統把描述轉成可渲染的共振結構，再輸出成遊戲與音樂工具可用的音效素材。

---

## 2. 項目目的

這個項目的目的，是建立一個低摩擦的聲音素材生成入口，幫助遊戲開發者、音效設計師與聲音工具開發者快速做出一整族同世界觀的小音效。

第一階段的重點放在：

- UI 點擊聲
- hover / confirm / cancel
- error / locked / unlock
- rare drop / item pickup
- magic one-shot
- short cue
- 環境提示聲
- 狀態轉換聲音

這些聲音不需要完整旋律，也不需要進入專業作曲流程。它們需要短、清楚、有情緒、有材質感，並且能大量產生變體。

這個工具的核心價值，是把「情緒」轉成「音色結構」，再把音色結構渲染成可用素材。

---

## 3. 核心洞見

遊戲裡很多聲音的情緒，並不一定來自旋律。

很多時候，情緒來自：

- 聲音亮不亮
- 尾音乾不乾淨
- 泛音是否穩定
- 高頻是否有金屬光
- 低頻是否有安全感
- 聲音是否一起呼吸
- 聲音是否逐漸失衡
- 不同泛音之間是否藏著秩序

因此，旋律可以保持簡單，情緒由音色承擔。

這個項目把聲音看成一組可控制的共振節點：

- 每個節點有頻率
- 每個節點有強度
- 每個節點有衰退時間
- 每個節點可以呼吸或擺動
- 節點之間可以形成比例關係
- 某些關係可以穩定
- 某些關係可以刻意錯開

這讓音色設計從「調旋鈕」提升成「設計聲音內部結構」。

## 4. 目標使用者

### 遊戲開發者

需要快速生成大量可用音效，例如按鈕、提示、獎勵、錯誤、解鎖、魔法、環境互動聲。

### 獨立遊戲團隊

需要同一世界觀下的一整族音效，但沒有足夠資源請專業音效團隊長時間製作。

### 音效設計師

可以用這個工具快速產生方向稿，再進一步加工。

### 電子音樂與合成器玩家

可以用共振結構做新的音色設計實驗。

### 聲音工具開發者

可以拿開源格式、參考渲染器與範例資料整合到自己的工具中。

## 5. 主要使用場景

### 場景 A：遊戲 UI 音效組

使用者輸入：

> 古代神殿、金屬、神聖、短促、乾淨、有一點稀有感。

系統輸出：

- hover.wav
- click.wav
- confirm.wav
- cancel.wav
- locked.wav
- unlock.wav
- rare_reward.wav
- resonance-profile.json

這些音效彼此可以不連續，但需要有共同聲音 DNA。

---

### 場景 B：情緒轉換 cue

使用者輸入：

> 一段 5 秒聲音，從懷疑慢慢變成恐懼。

系統渲染：

- 低頻支撐逐漸變弱
- 高頻變亮
- 泛音偏移增加
- 呼吸逐漸不同步
- 金屬噪聲增加
- 尾音變得不穩

輸出：

- suspicion_to_fear.wav
- suspicion_to_fear_profile.json

---

### 場景 C：魔法 one-shot

使用者輸入：

> 一個短的治癒魔法聲，溫暖、乾淨、帶一點晶體光。

系統渲染：

- 短 attack
- 中長尾音
- 5 系比例增加甜感
- 高頻少量閃光
- 噪聲降低
- 呼吸保持穩定

輸出：

- heal_magic_01.wav
- heal_magic_02.wav
- heal_magic_soft.wav

- 強度
- 衰退

- 相位

這個格式是項目的核心資產。

### 3. 參考渲染器

提供一個簡單可靠的渲染器，把 profile 轉成 WAV。

- additive 渲染
- modal 渲染
- noise layer
- simple reverb
- basic automation

## 7. 核心工作流

```text
情緒 / 場景 / 材質描述
→ 音色意圖
→ 共振結構
→ 渲染方向
→ 試聽
→ 調整
→ 匯出 WAV / JSON
```

更底層的資料流程：

```text
頻譜曲線
→ peaks
→ partial nodes
→ relation graph
→ resonance profile
→ renderer
→ game-ready audio asset
```

這條流程同時支援兩個方向：

1. 從真實聲音分析出 profile
2. 從情緒描述生成新的 profile

## 8. 音色設計語言

這個項目使用一組簡單、可被人理解的音色特徵。

### 基礎特徵

| 特徵 | 意義 |
|---|---|
| 頻率位置 | 聲音的高低與亮度基礎 |
| 強度 | 哪些成分比較突出 |
| 衰退時間 | 聲音短促、乾淨、空靈或拖尾 |
| 呼吸速度 | 聲音是否有脈動感 |
| 呼吸深度 | 脈動是否明顯 |
| 相位錯開 | 聲音是否有旋轉、散開、不穩定感 |
| 高頻比例 | 閃光、尖銳、金屬、空氣感 |
| 噪聲比例 | 摩擦、腐化、粗糙、攻擊性 |
| 比例關係 | 聲音內部是否有秩序 |

### 關係特徵

聲音裡的泛音可以形成關係網。

關係網不是單純看線多不多，而是看線出現在哪裡。

有價值的結構包括：

- 主音有一條穩定錨點
- 高階泛音形成隱藏對齊
- 5 系提供甜感與溫度
- 7 系提供古怪、金屬、儀式感
- 低階避開太普通的 2 倍、3 倍
- 高階藏著漂亮比例
- 呼吸週期有比例但相位錯開

這些結構讓聲音有「奇怪但不亂」的質感。

## 9. 兩種主要模式

### Palette Mode：音效家族模式

產生一組離散但同族的音效。

適合：

- UI set
- item set
- magic set
- temple set
- corrupted set
- rare reward set

這種模式不要求音色彼此連續。  
重點是它們共享同一套材質、空間、共振語法與情緒方向。

---

### Trajectory Mode：情緒軌跡模式

產生一段會變化的聲音。

適合：

- 懷疑 → 恐懼
- 開心 → 興奮
- 安全 → 危險
- 乾淨 → 腐化
- 普通 → 稀有
- 低能量 → 充能完成

這種模式的重點是連續微調音色，而不是寫複雜旋律。

旋律可以簡單，情緒變化由音色承擔。

## 14. 長期願景

長期來看，這個項目可以成為一個共振結構音色設計入口。

它的願景不是取代作曲家，也不是做一鍵生成完整歌曲。

它要提供的是一種新的聲音設計方式：

```text
描述情緒
→ 設計共振結構
→ 生成音效家族
→ 控制情緒軌跡
→ 匯出到遊戲與音樂工具
```

最有價值的方向，是讓遊戲開發者不用進入完整作曲流程，也能做出有情緒、有材質、有世界觀的小聲音。

這個項目最終可以長成：

- 共振音效生成器
- 情緒驅動 SFX 工具
- 遊戲聲音素材生成器
- 開源音色結構格式
- 聲音工具的參考實作
- 一套可被其他音樂軟體吸收的設計語言

核心價值是：

> 把音色裡的情緒、秩序與材質，變成可以描述、可以調整、可以渲染、可以分享的結構。
